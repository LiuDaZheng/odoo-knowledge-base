# CRM 最佳实践

Odoo CRM 使用最佳实践指南。

## 📋 线索管理最佳实践

### 1. 线索录入规范

**必填信息**:
- ✅ 线索名称（清晰描述）
- ✅ 联系人姓名
- ✅ 联系方式（电话或邮箱至少一项）
- ✅ 公司名称
- ✅ 线索来源

**推荐信息**:
- 📌 预计收入
- 📌 优先级
- 📌 标签分类
- 📌 初步需求描述

**示例**:
```
✅ 好：某某公司 - ERP 系统升级项目 - 张三 - 138xxxx
❌ 差：潜在客户 - 无具体信息
```

### 2. 线索分级标准

**A 级（高优先级）**:
- 预算已确认
- 决策链清晰
- 需求紧急（3 个月内）
- 预计收入 > 50 万

**B 级（中优先级）**:
- 预算审批中
- 需求明确（6 个月内）
- 预计收入 20-50 万

**C 级（低优先级）**:
- 预算未确定
- 需求模糊（6 个月以上）
- 预计收入 < 20 万

### 3. 线索跟进节奏

| 线索级别 | 首次联系 | 跟进频率 | 最长周期 |
|---------|---------|---------|---------|
| A 级 | 24 小时内 | 每 2-3 天 | 30 天 |
| B 级 | 48 小时内 | 每周 1-2 次 | 60 天 |
| C 级 | 1 周内 | 每 2 周 1 次 | 90 天 |

### 4. 线索转化时机

**转化信号**:
- ✅ 客户明确表达购买意向
- ✅ 预算已获批
- ✅ 决策人参与沟通
- ✅ 需求调研完成
- ✅ 技术方案确认

**转化检查清单**:
- [ ] 客户需求文档
- [ ] 预算确认邮件/文件
- [ ] 决策链图谱
- [ ] 竞争对手分析
- [ ] 初步方案大纲

## 📋 机会管理最佳实践

### 1. 机会创建规范

**必需信息**:
- ✅ 机会名称（客户 + 项目）
- ✅ 关联客户
- ✅ 预计收入
- ✅ 成功概率
- ✅ 当前阶段
- ✅ 销售负责人
- ✅ 预计成交日期

**示例**:
```
✅ 好：某某科技 - CRM 系统实施项目 - 30 万 - 60% - 2026-06-30
❌ 差：新机会 - 无客户关联
```

### 2. 概率评估标准

| 概率 | 阶段特征 | 关键条件 |
|------|---------|---------|
| 10% | 初步接触 | 已建立联系，需求待确认 |
| 30% | 需求确认 | 需求明确，预算待确认 |
| 50% | 方案演示 | 方案认可，进入商务阶段 |
| 70% | 报价谈判 | 价格谈判中，条款待确认 |
| 90% | 合同审批 | 合同审核中，等待签约 |
| 100% | 赢单 | 已签约 |

### 3. 机会推进要求

**每个阶段必须完成**:

**新线索 → 已联系**:
- [ ] 电话/邮件联系
- [ ] 确认联系人身份
- [ ] 记录初步需求

**已联系 → 需求确认**:
- [ ] 需求调研会议
- [ ] 需求文档确认
- [ ] 预算范围确认

**需求确认 → 方案演示**:
- [ ] 解决方案设计
- [ ] 产品演示安排
- [ ] 技术答疑完成

**方案演示 → 报价谈判**:
- [ ] 正式报价单
- [ ] 商务条款确认
- [ ] 竞争对手分析

**报价谈判 → 赢单**:
- [ ] 合同定稿
- [ ] 内部审批完成
- [ ] 客户签约

### 4. 输单分析

**必须记录输单原因**:
- 价格过高
- 功能不满足
- 竞争对手优势
- 预算不足
- 项目延期/取消
- 决策人变更
- 其他（具体说明）

**输单复盘**:
```markdown
## 输单复盘：某某公司项目

**基本信息**:
- 机会金额：50 万
- 跟进周期：3 个月
- 竞争对手：XX 软件

**输单原因**:
价格过高（客户反馈我们的报价比竞争对手高 30%）

**改进措施**:
1. 优化定价策略
2. 加强价值传递
3. 提前了解竞争对手报价

**经验教训**:
- 早期应了解客户预算范围
- 需要更清晰传达产品差异化价值
```

## 📋 客户管理最佳实践

### 1. 客户信息完整性

**公司客户必填**:
- ✅ 公司名称
- ✅ 统一社会信用代码
- ✅ 公司地址
- ✅ 官方网站
- ✅ 主营业务
- ✅ 公司规模

**联系人必填**:
- ✅ 姓名
- ✅ 职位
- ✅ 手机/电话
- ✅ 邮箱
- ✅ 微信（如有）

### 2. 客户分类标签

**按行业**:
- 制造业
- 零售业
- 服务业
- 互联网
- 金融
- 教育
- 医疗

**按规模**:
- 大型企业 (>1000 人)
- 中型企业 (100-1000 人)
- 小型企业 (<100 人)

**按价值**:
- VIP 客户 (年消费>100 万)
- 重点客户 (年消费 50-100 万)
- 普通客户 (年消费<50 万)

**按区域**:
- 华北区
- 华东区
- 华南区
- 西部区

### 3. 客户互动记录

**每次互动必须记录**:
- 📌 互动日期
- 📌 互动方式（电话/会议/邮件/微信）
- 📌 参与人员
- 📌 讨论要点
- 📌 下一步行动
- 📌 负责人
- 📌 截止日期

**示例**:
```markdown
## 2026-04-12 电话跟进

**参与人员**: 张三（我方）、李四（客户采购经理）

**讨论要点**:
1. 确认项目预算已获批（50 万）
2. 客户希望 5 月中旬开始实施
3. 需要增加移动端功能

**下一步行动**:
- [ ] 更新方案，增加移动端功能（王五，4/15）
- [ ] 准备合同草案（赵六，4/18）
- [ ] 安排技术答疑会议（张三，4/20）
```

### 4. 客户健康度评估

**健康客户指标**:
- ✅ 定期互动（每月至少 1 次）
- ✅ 有新需求或增购意向
- ✅ 付款及时
- ✅ 满意度高
- ✅ 愿意推荐

**风险客户信号**:
- ⚠️ 长期无互动（>3 个月）
- ⚠️ 投诉增多
- ⚠️ 付款延迟
- ⚠️ 关键联系人离职
- ⚠️ 竞争对手接触

## 📋 销售管道最佳实践

### 1. 管道可视化

**每日检查**:
- 查看管道漏斗图
- 识别瓶颈阶段
- 关注即将到期机会
- 检查新增线索

**每周分析**:
- 各阶段转化率
- 平均销售周期
- 团队业绩对比
- 预测准确率

### 2. 管道清理

**每周清理**:
- 关闭已赢单机会
- 关闭已输单机会
- 更新停滞机会状态
- 删除无效线索

**每月清理**:
- 归档 90 天无更新线索
- 重新分配长期未跟进线索
- 清理重复客户信息

### 3. 预测管理

**预测准确性提升**:
- 每周更新机会概率
- 基于历史数据校准
- 考虑季节性因素
- 区分承诺预测和最佳预测

**预测分类**:
```
承诺预测 (Commit): 90% 以上概率，已确认
最佳预测 (Best Case): 50-90% 概率，可能成交
管道预测 (Pipeline): 50% 以下概率，潜在机会
```

## 📋 团队协作最佳实践

### 1. 线索分配

**分配原则**:
- 按行业专长分配
- 按区域分配
- 按工作量平衡
- 按线索来源分配（官网/市场活动/转介绍）

**分配后跟进**:
- 24 小时内必须联系
- 48 小时内更新状态
- 7 天内评估是否继续

### 2. 交接规范

**销售内部交接**:
- [ ] 完整客户信息
- [ ] 所有沟通记录
- [ ] 当前进展状态
- [ ] 下一步计划
- [ ] 风险提示

**销售到实施交接**:
- [ ] 合同副本
- [ ] 客户需求文档
- [ ] 承诺功能清单
- [ ] 关键干系人
- [ ] 项目时间表

### 3. 知识共享

**成功案例库**:
- 行业解决方案
- 典型客户案例
- 竞争对手分析
- 价格策略参考

**失败案例库**:
- 输单原因分析
- 竞争对手优势
- 改进建议
