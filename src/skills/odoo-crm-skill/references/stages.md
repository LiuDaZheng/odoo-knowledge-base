# CRM 管道阶段配置

销售管道阶段定义和配置指南。

## 标准管道阶段

### 默认阶段序列

| 序号 | 阶段名称 | 英文 | 概率范围 | 描述 |
|------|---------|------|---------|------|
| 1 | 新线索 | New | 0-10% | 刚创建的线索，尚未联系 |
| 2 | 已联系 | Contacted | 10-20% | 已初步联系客户 |
| 3 | 需求确认 | Qualified | 20-40% | 确认客户需求和预算 |
| 4 | 方案演示 | Proposition | 40-60% | 提供解决方案并演示 |
| 5 | 报价谈判 | Proposal/Quotation | 60-80% | 发送报价并谈判 |
| 6 | 赢单 | Won | 100% | 成功签约 |
| 7 | 输单 | Lost | 0% | 未成功签约 |

## 阶段配置

### 添加自定义阶段

```bash
# 通过界面配置
设置 → 销售 → CRM → 阶段 → 创建

# 通过 API 配置
python3 {baseDir}/scripts/crm.py pipeline add-stage \
  --name "技术评估" \
  --sequence 3 \
  --probability 30 \
  --fold false \
  --on_hold false
```

### 阶段字段说明

| 字段 | 类型 | 描述 |
|------|------|------|
| name | char | 阶段名称 |
| sequence | integer | 排序序号 |
| probability | float | 默认成功概率 |
| fold | boolean | 是否在管道中折叠 |
| on_hold | boolean | 是否为暂停阶段 |
| team_id | many2one | 关联的销售团队 |

## 阶段转换规则

### 有效转换

| 从阶段 | 到阶段 | 说明 |
|--------|--------|------|
| 新线索 | 已联系 | ✅ 正常推进 |
| 已联系 | 需求确认 | ✅ 正常推进 |
| 需求确认 | 方案演示 | ✅ 正常推进 |
| 方案演示 | 报价谈判 | ✅ 正常推进 |
| 报价谈判 | 赢单 | ✅ 成功 |
| 任何阶段 | 输单 | ✅ 失败关闭 |

### 无效转换（需审批）

| 从阶段 | 到阶段 | 说明 |
|--------|--------|------|
| 新线索 | 赢单 | ⚠️ 跳过所有阶段 |
| 新线索 | 输单 | ⚠️ 未尝试联系 |
| 报价谈判 | 需求确认 | ⚠️ 倒退阶段 |

## 阶段 KPI 指标

### 转化率

```
阶段转化率 = (下一阶段机会数 / 当前阶段机会数) × 100%

示例:
- 新线索 → 已联系：80%
- 已联系 → 需求确认：60%
- 需求确认 → 方案演示：50%
- 方案演示 → 报价谈判：70%
- 报价谈判 → 赢单：40%

整体转化率 = 80% × 60% × 50% × 70% × 40% = 6.72%
```

### 平均停留时间

| 阶段 | 目标天数 | 预警天数 |
|------|---------|---------|
| 新线索 | 1-2 天 | >3 天 |
| 已联系 | 2-3 天 | >5 天 |
| 需求确认 | 3-5 天 | >7 天 |
| 方案演示 | 5-7 天 | >10 天 |
| 报价谈判 | 7-14 天 | >21 天 |

### 管道健康度

```
健康管道特征:
- 各阶段机会数呈金字塔形（上宽下窄）
- 无机会在单一阶段停留过久
- 赢单率稳定在 20-40%
- 平均销售周期符合预期

预警信号:
- 某阶段机会数异常集中
- 大量机会接近截止日期
- 赢单率持续低于 15%
- 销售周期明显延长
```

## 自动化规则

### 阶段自动推进

```python
# 示例：满足条件自动推进阶段
条件:
  - 客户确认预算
  - 完成技术评估
  - 收到采购意向书

动作:
  - 推进到下一阶段
  - 通知销售经理
  - 更新预计成交日期
```

### 逾期预警

```python
# 示例：阶段停留超时预警
规则:
  - 新线索 > 3 天未联系 → 发送提醒
  - 报价阶段 > 14 天 → 升级处理
  - 所有阶段 > 30 天 → 经理介入

通知方式:
  - 站内消息
  - 邮件通知
  - 钉钉/企业微信
```
