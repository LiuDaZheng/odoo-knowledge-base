# Odoo Skill 质量核查标准

参考 XR-Frame Skill 核查标准制定，总分 100 分。

## 一、完整性（25 分）

### 1.1 文件结构完整性（10 分）
- [ ] **必需文件存在** (4 分)
  - SKILL.md 存在且非空 (2 分)
  - references/ 目录存在 (1 分)
  - 至少有 1 个参考文档 (1 分)

- [ ] **目录结构规范** (3 分)
  - 技能名称使用 kebab-case (1 分)
  - 目录层级清晰 (1 分)
  - 无多余文件 (1 分)

- [ ] **辅助资源** (3 分)
  - scripts/ 目录（如有脚本）(1 分)
  - src/ 目录（如有源码）(1 分)
  - 元数据文件（_meta.json 或 .clawhub）(1 分)

### 1.2 内容完整性（15 分）
- [ ] **YAML Frontmatter** (5 分)
  - name 字段存在且唯一 (2 分)
  - description 字段存在且 < 100 字符 (2 分)
  - 无其他必需字段缺失 (1 分)

- [ ] **核心章节** (6 分)
  - 快速导航/目录 (2 分)
  - 主体内容完整 (2 分)
  - 参考文档链接有效 (2 分)

- [ ] **参考文档** (4 分)
  - 每个主题有对应参考文档 (2 分)
  - 参考文档内容充实 (2 分)

---

## 二、可用性（25 分）

### 2.1 触发器清晰度（10 分）
- [ ] **触发场景描述** (5 分)
  - 清晰说明何时使用该 Skill (3 分)
  - 触发关键词明确 (2 分)

- [ ] **使用示例** (5 分)
  - 提供至少 1 个使用示例 (3 分)
  - 示例覆盖常见场景 (2 分)

### 2.2 导航与检索（10 分）
- [ ] **快速导航** (5 分)
  - 目录/导航表清晰 (3 分)
  - 内部链接有效 (2 分)

- [ ] **搜索友好** (5 分)
  - 关键词在 description 中体现 (3 分)
  - 标题层级清晰 (H1/H2/H3) (2 分)

### 2.3 代码与示例（5 分）
- [ ] **代码示例** (3 分)
  - 提供相关代码示例 (2 分)
  - 代码格式正确 (1 分)

- [ ] **示例可执行性** (2 分)
  - 示例可独立运行/理解 (2 分)

- [ ] **文件命名** (2 分)

  - 目录命名规范 (1 分)

### 3.3 文档规范（5 分）
